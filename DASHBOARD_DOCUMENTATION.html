5 Key Calculations (Frontend)

5.1 — Output Pieces

Determined by efficiencyData.flags. Whichever section flag = 1, its Line-level pcs (regular + OT) are summed.

FlagSource FieldLabel Shown
I_L_Output_Sewing = 1LineSWGSewnPcs + LineSWGSewnPcsOTSewing
I_L_Output_QC = 1LineCHKPassedPcs + LineCHKPassedPcsOTEndLine Checking
I_L_Output_AQL = 1LineAQLPassedPcs + LineAQLPassedPcsOTAQL
I_L_Output_Finishing = 1LineFINPassedPcs + LineFINPassedPcsOTFinishing
I_L_Output_Packing = 1LinePKGPassedPcs + LinePKGPassedPcsOTPacking
No flags set (fallback)LineSWGSewnPcs + LineSWGSewnPcsOTSewing

5.2 — Target vs Actual Efficiency (taEff)

SameDate (live shift)
fCurrTargetPcs = (LineTargetPcs / ShiftTime) × AvailableMinsWithBreak
NotSameDate (historical)
fCurrTargetPcs = LineTargetPcs (full)
Result
taEff = (outputPcs / fCurrTargetPcs) × 100

5.3 — Efficiency % (grsEff / onsEff)

Prefers backend value
if efficiencyData.efficiency > 0 → use it directly
Fallback (frontend)
grsEff = totalEarnedMinutes / totalWorkedMins × 100
onsEff = totalEarnedMinutes / (totalWorkedMins − offStandardMins) × 100

5.4 — OEE Calculations NEW

MetricFormulaSource Fields
ShiftTimeTillNow SameDate → AvailableMinsWithBreak
NotSameDate → ShiftTime		 efficiencyData.shiftDetails
Availability totalWorkedMins / (shiftTimeTillNow × activeWS) × 100 All section WorkedMins summed
Performance totalEarnedMinutes / totalWorkedMins × 100 All section PcsSAM / WorkedMins
Quality SectionCHKPassedPcs / SectionCHKLoadingPcs × 100 EndLine CHK passed vs loaded
OEE Score (Availability/100) × (Performance/100) × (Quality/100) × 100 Composite

5.5 — DownTime Analysis Percentages

Clocked Minutes = ManDayMinutes × activeWS
Each category % = (categoryMins / ClockedMinutes) × 100

5.6 — Derived Values

VariableFormula
totalWSSum of CapacityWs across all sections (SWG+CHK+AQL+FIN+PKG)
activeWSSum of ActualWs across all sections
prsOprsCurrNoOfOprsLoggedIn + CurrNoOfQcOprsLoggedIn
attRate(prsOprs / activeWS) × 100
targetOutputPerHrLineTargetPcs / LineNoOfHrs
actualOutputPerHr(outputPcs / ManDayMinutes) × 60
estOutputSameDate: outputPcs + ((shiftTime − elapsedMins) × (outputPcs / elapsedMins))
productivityPerPersonoutputPcs / manDays

GET /api/dashboard/top-operators Top & Low Performing Operators — version-aware query
DBTablesNote
transDBT_EmpLoginDt_{Month}PcsSam + AvailMins per operator
masterDBM_EmployeeName + EmployeeNo (Active='Y' only)
transDBT_ProductionDt_{Month} + M_OperationOperation names (STRING_AGG or FOR XML PATH)
Eff% per operator = SUM(PcsSam) / SUM(AvailMins) × 100
Top 5 = first 5 (highest Eff)  |  Bottom 5 = last 5 from remaining (reversed)

SQL Server version check: >= 2017 uses STRING_AGG; older uses STUFF + FOR XML PATH.

GET /api/dashboard/top-rw-operators Top 5 Rework Operators ★ NEW
DBTablesFilter
transDBT_QcProductionDt_{Month}ProcessCode = '004' (EndLine CHK)
masterDBM_QcOperationSeq, M_QcOperationJOIN to get ProcessCode
masterDBM_EmployeeLEFT JOIN for name/number
SELECT TOP 5
  ISNULL(e.EmployeeName, a.EmployeeCode) AS EmployeeName,
  ISNULL(e.EmployeeNo, '')               AS EmployeeNo,
  SUM(a.ReWorkPcs)  AS ReworkPcs,
  SUM(a.CheckedPcs) AS CheckedPcs,
  CASE WHEN SUM(a.CheckedPcs) > 0
       THEN ROUND(SUM(a.ReWorkPcs) * 100.0 / SUM(a.CheckedPcs), 1)
       ELSE 0 END AS RwPct
FROM [transDB]..[T_QcProductionDt_{Month}] a
INNER JOIN [masterDB]..M_QcOperationSeq b ON ...
INNER JOIN [masterDB]..M_QcOperation c ON c.ProcessCode = '004'
LEFT JOIN  [masterDB]..M_Employee e ON a.EmployeeCode = e.EmployeeCode
WHERE CAST(a.ProdnDate AS DATE) = CONVERT(DATE, @qdate, 103)
  AND a.ShiftCode = @shiftCode AND a.LineCode = @lineCode
GROUP BY a.EmployeeCode, e.EmployeeName, e.EmployeeNo
HAVING SUM(a.ReWorkPcs) > 0
ORDER BY SUM(a.ReWorkPcs) DESC

GET /api/dashboard/wip-report WIP operation-wise chart modal — 4 steps, most complex report
StepTablesWhat it gets
1T_FactLineProductionStyleCodeStg for active styles
2 (loop per style)M_OperationSeq, M_Operation, M_GroupAll operations in order
3T_Tags, M_StyleHd, V_StylePoHd, M_OperationSeqBundle location data (COprCodeStg, COprPcsStg)
4 (Node.js)—WIP placement algorithm
WIP Placement Rule
If cPcs[LastOprSeqNo op] >= BundlePcs → bundle FINISHED → place at NEXT op
Otherwise → bundle IS AT that op → place at SAME op
WIP Pcs = BundlePcs − ProdnPcs (partial pieces already done at placement op)
